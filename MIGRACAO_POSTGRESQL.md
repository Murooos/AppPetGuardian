# 🐘 Migração para PostgreSQL no Render.com

Este documento explica como configurar o PostgreSQL no Render.com após a migração do Oracle.

## ✅ Alterações Realizadas

1. ✅ `requirements.txt` - Substituído `oracledb` por `psycopg2-binary`
2. ✅ `gerenciador_conexão.py` - Reescrito para usar PostgreSQL
3. ✅ `scrips.py` - Atualizado para sintaxe PostgreSQL (`%(param)s` em vez de `:param`)
4. ✅ `schema_postgresql.sql` - Schema convertido do Oracle para PostgreSQL
5. ✅ `render.yaml` - Simplificado, removidas configurações do Oracle

## 📋 Passos para Configurar no Render.com

### 1. Criar Banco PostgreSQL no Render

1. Acesse: https://dashboard.render.com
2. Clique em **"New +"** → **"PostgreSQL"**
3. Configure:
   - **Name**: `pet-guardian-db` (ou outro nome - **IMPORTANTE**: use o mesmo nome no `render.yaml`)
   - **Database**: `petguardian`
   - **User**: `petguardianuser`
   - **Region**: `Oregon` (mesma região do serviço web)
   - **PostgreSQL Version**: `16` (ou mais recente)
   - **Plan**: `Free`
4. Clique em **"Create Database"**

### 2. Configurar Variáveis de Ambiente no Serviço Web

1. No dashboard do Render, abra seu serviço web (`pet-guardian-api`)
2. Vá em **"Environment"**
3. Adicione/verifique as variáveis:

```
DATABASE_URL=postgresql://petguardianuser:xc515UQKromm7JraX8M76YafzI6hhLyX@dpg-d497beemcj7s73ebag6g-a/petguardian
JWT_SECRET_KEY=sua_chave_secreta_aqui
ALLOWED_ORIGINS=*
```

**OU** use a opção "Link Database" no Render para conectar automaticamente.

### 3. Criar as Tabelas no PostgreSQL

Após criar o banco, você precisa executar o schema SQL:

1. No dashboard do Render, abra o banco PostgreSQL criado
2. Vá em **"Connections"**
3. Copie o **"PSQL Command"** ou use a **"External Database URL"**
4. Execute o arquivo `Backend/Banco_dados/schema_postgresql.sql` no banco

**Opção 1: Via psql (linha de comando)**
```bash
# Use o comando PSQL fornecido pelo Render
PGPASSWORD=senha psql -h host -U usuario -d database -f Backend/Banco_dados/schema_postgresql.sql
```

**Opção 2: Via Render Shell**
1. No dashboard do banco, clique em **"Connect"** → **"Shell"**
2. Execute o conteúdo do arquivo `schema_postgresql.sql`

**Opção 3: Via Script Python (recomendado)**
Crie um script temporário para executar o schema:

```python
import psycopg2
import os
from urllib.parse import urlparse

database_url = "postgresql://petguardianuser:xc515UQKromm7JraX8M76YafzI6hhLyX@dpg-d497beemcj7s73ebag6g-a/petguardian"

parsed = urlparse(database_url)
conn = psycopg2.connect(
    host=parsed.hostname,
    port=parsed.port or 5432,
    database=parsed.path[1:],
    user=parsed.username,
    password=parsed.password,
    sslmode='require'
)

with open('Backend/Banco_dados/schema_postgresql.sql', 'r', encoding='utf-8') as f:
    schema = f.read()
    
cursor = conn.cursor()
cursor.execute(schema)
conn.commit()
cursor.close()
conn.close()

print("✅ Schema criado com sucesso!")
```

### 4. Verificar o render.yaml

Certifique-se de que o nome do banco no `render.yaml` corresponde ao nome do banco criado no Render:

```yaml
- key: DATABASE_URL
  fromDatabase:
    name: pet-guardian-db  # ← Deve ser o mesmo nome do banco criado
    property: connectionString
```

Se o nome for diferente, atualize o `render.yaml` ou renomeie o banco no Render.

### 5. Fazer Deploy

1. Faça commit das alterações:
```bash
git add .
git commit -m "Migração para PostgreSQL"
git push
```

2. O Render fará o deploy automaticamente
3. Verifique os logs para confirmar que a conexão foi estabelecida

## 🐛 Troubleshooting

### Erro: "DATABASE_URL não configurada"

**Solução**: Verifique se a variável `DATABASE_URL` está configurada no Render (Environment Variables).

### Erro: "relation does not exist"

**Solução**: Execute o schema SQL (`schema_postgresql.sql`) no banco de dados.

### Erro: "could not connect to server"

**Solução**: 
- Verifique se o banco está rodando no Render
- Verifique se está usando a URL correta (Internal ou External)
- Para serviços no mesmo Render, use a **Internal Database URL**

### Erro: "SSL connection required"

**Solução**: O código já está configurado com `sslmode='require'`. Se ainda houver erro, verifique se a URL está correta.

## 📝 Diferenças Oracle vs PostgreSQL

| Oracle | PostgreSQL |
|--------|------------|
| `NUMBER` | `INTEGER` ou `SERIAL` |
| `VARCHAR2` | `VARCHAR` |
| `CLOB` | `TEXT` |
| `NUMBER(1)` | `BOOLEAN` |
| `:param` | `%(param)s` |
| `ativo = 1` | `ativo = TRUE` |
| `GENERATED BY DEFAULT AS IDENTITY` | `SERIAL` |
